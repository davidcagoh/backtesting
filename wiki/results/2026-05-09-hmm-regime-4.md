## Read

**Win-rate hypothesis confirmed.** Open hypothesis #5 in `wiki/learnings.md` predicted HMM regime detection would lift win rate from ~22% (SmaRegime180) toward 40%+. Result: 45.9%. Mechanism appears to be working — entering only when P(bull-state) > 0.65 produces a cleaner directional signal than the SMA slope gate.

**Profit Factor went the wrong way** — 2.72 → 1.58. HmmRegime4 wins more often but each win is smaller relative to the losses. Avg winner duration 2d 02:41 vs avg loser 0d 19:26 suggests the exit signal is firing fast on losers (good — cuts losses) but holding winners only modestly. Total return ends up roughly equal to SmaRegime180 despite double the trades and double the win rate.

**Calmar 26 is structurally inflated.** Max DD is 1.03% — extremely small denominator. A single 2% drawdown would halve Calmar. SQN 1.38 (vs SmaRegime180's 1.02) is the more honest improvement signal — modest but real, and at N=74 it's statistically meaningful (vs N=32).

**Look-ahead caveat (per strategy docstring):** the HMM is fit on the full visible window, so this is an upper bound on signal quality, not a clean OOS result. To make this an honest comparison would require rolling-window HMM refit (re-fit every N bars on prior data only). Result should be read as "best case if HMM regime structure is stable enough to survive look-ahead."
